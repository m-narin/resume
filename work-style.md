## コミュニケーション

- リモートワークする機会も多い職種なので円滑に気持ちよく仕事を進められることを常に心がけています。（HRTの意識）
- ミーティングを開く際は事前準備して内容を共有し、意思決定やネクストアクションを決めるために効率よく実施する意識を持っています。
- テキストコミュニケーションでは、感情が伝わるように絵文字や感嘆符を積極的に使います。

## 開発スタイル

- Pull Requestは、サイクルタイム向上やレビュー負荷軽減の観点から小さく丁寧に出す意識を持っています。
- 生成AIは積極的に活用しますが、PRを提出する前にすべての内容が説明できることを確認しています。
- 開発生産性向上のための工夫は日頃から取り入れるようにしています。（例: [Claude Code rules](.claude/rules/), [skills](.claude/skills/)）

## 自己研鑽・基礎力

- 生成AIによる出力を正しく判断し、より良いものにするため毎月1冊は技術書を読んで自身の基礎力向上に投資しています。
- 基礎学習を大切にしており、社会人1年目にCSの基礎（アルゴリズムとデータ構造）の勉強会を同期と開催していました。後に[本形式にまとめて公開](https://zenn.dev/mandenaren/books/algorithm-study)しています。
- 睡眠時間や運動習慣、プライベートも大事にしており、結果として中長期の安定した稼働（=健康）や関係性に繋がると考えています。

## 利用ツール

- Claude Code

- Slack: 仕事用
- Rectangle: マウス操作で容易に画面分割できる
- Clipy: コピー履歴を辿れる

## 早期キャッチアップの工夫

前回転職した際に実施したことは、

- 入社前にキャッチアップが必要な技術をヒアリングし自己学習、入社直後からIssueを実行できるようにした。
- オンボーディング期間は、頻度高く分報をあげながらタスクを進め、管理コスト・手戻りや認識違いを最小化した。

## Macの設定など

- ポインターのサイズや色を見やすいようにする(画面共有で説明する際に、大きくてオレンジ色や緑色だと見えやすい)
- 全角の数字は半角数字に自動変換する
- CapsLockは無効化する
- shellのaliasを活用し、コマンドをショートカットする
- ユーザー辞書で入力をショートカットする
